HTML

  • HTML creates content structure for a webpage.
  • The head element contains information about the webpage.
  • The head contains info that helps the browser render the page correctly. Not visible to users.
  • Browsers use metadata to effectively serve the webpage to users.
  • The body element represents the visible content shown to the user.
  • It's important to use headers in number order for accessibility.
  • HTML attributes define additional info about elements. They are defined in the beginning element, never the closing.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three ways to style a webpage using CSS; an external CSS style sheet, inline CSS, and an internal CSS style sheet.
  • External CSS style sheets are the most common ways to style webpages.
  • The selector starts off CSS code and defines the element or attributes to which the rules or declarations apply.
  • The * selector apply's rules to all elements visible on a page.
  • Declarations contain 2 important components: the CSS property applied and the value of the property.
  • Class attribute allows us to share CSS rules to any element we choose by assigning rules to class attribute with the class selector.
  • Assigning colors in CSS can be done in many ways, including semantic references (gray, blue) or hex #.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git init: initializes a new git repository.
  • git status: shows the status of the current repository.
  • git commit: saves changes to git's repository.
  • git add -A: staging area informs git thath changes will be added in next commit.
  • git pull origin main: pulls down changes from remote branh to synce with local branch.
